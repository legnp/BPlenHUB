# Mapa 1 — Features e Variações

Mapeia toda feature "pai" do sistema e suas variações, contando quantas vezes
cada variação é efetivamente entregue/renderizada no projeto (a mesma feature
aparecendo em páginas diferentes conta uma vez por página onde aparece).

Ainda não populado — a ser preenchido pelo chat de planejamento a partir de
inspeção real do código (não assumir a partir de nomes de arquivo).

## Formato esperado

```
- Feature pai: Agendamento de Eventos
  - Variação: Agendamento 1-to-1 → entregue em: [página GDC, página Gestão de Carreira] (2x)
  - Variação: Devolutiva Comportamental → entregue em: [página X] (1x)
  - Variação: Devolutiva de Plano de Carreira → entregue em: [página Y] (1x)
```
